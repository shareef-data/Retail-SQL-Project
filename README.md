# Retail Sales SQL Project

## Author
*Shareef*

## Project Overview
This project simulates a *retail sales database* using MySQL. It demonstrates database creation, data insertion, and advanced business analysis queries to generate insights for decision-making. The project includes tables for products, customers, orders, and order items.

## Tables Created
- *Products* – stores product information (ID, name, category, price)  
- *Customers* – stores customer details (ID, name, email, region, join date)  
- *Orders* – stores order information (order ID, customer ID, order date)  
- *Order_Items* – stores details of products in each order (order ID, product ID, quantity)

## Business Analysis Queries
The project includes queries to answer key business questions such as:

1. *Total Revenue* – Calculate total revenue generated from all orders  
2. *Monthly Revenue* – Extract month-wise revenue trends  
3. *Region-wise Revenue* – Analyze which regions generate the most revenue  
4. *Top Customers* – Identify customers contributing the most to revenue  
5. *Repeat vs One-Time Customers* – Analyze customer retention  
6. *Revenue Contribution by Product* – Determine the percentage of total revenue contributed by each product  

## Key Insights
- Helps identify *best-selling products* and high-revenue categories  
- Supports *customer retention analysis* by classifying one-time and repeat buyers  
- Provides *region-wise and month-wise revenue trends* for marketing decisions  
- Highlights products contributing the most to total revenue  
- Serves as a *portfolio-ready project* demonstrating SQL skills and business analysis capability
